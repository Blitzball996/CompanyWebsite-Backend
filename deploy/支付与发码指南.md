# 付款自动发序列号 — 怎么用（大白话）

> 目标：用户在你网站付钱 → 系统**自动**生成序列号 → **自动**邮件发给他。全程不用你手动操作。

---

## 整个流程长这样

```
用户点 pricing 页「购买 Pro」
   → 跳转到 Airwallex 收银台付款
   → 付款成功
   → Airwallex 自动通知你的后端 (webhook)
   → 后端验证通知真实性 → 生成序列号 → 存库 → 邮件发给买家
   → 用户收到邮件里的序列号 → 在软件里激活
```

你只需要把下面几样**配置好**，剩下全自动。

---

## 你要做的（一次性配置）

### 1. 注册 Airwallex 商户（只有你能做）
拿公司证件去 airwallex.com 注册，开通收款。

### 2. 创建收银台 / 支付链接
在 Airwallex 后台为每个产品建支付链接（Blitz Pro、CloseCrab Pro…）。
**关键**：创建时在「metadata / 自定义字段」里填两个值，后端靠它判断发哪个产品的码：
```
product = blitz-pro      (或 blitz-standard / closecrab-pro / closecrab-standard)
email   = {买家邮箱}      (能用 Airwallex 的客户邮箱变量就自动带入)
```
把生成的支付链接贴到前端 `pricing.html` 顶部的 `PAY_LINKS` 里。

### 3. 配置 Webhook（让 Airwallex 能通知你后端）
在 Airwallex 后台 → Webhooks，添加一个回调地址：
```
https://blitzball.lol/api/webhook/airwallex
```
设置一个**签名密钥（secret）**，把同样的值填到后端 `.env` 的 `WEBHOOK_SECRET`。
（这个 secret 用来验证"通知确实来自 Airwallex"，防伪造。）

### 4. 配置发邮件（SMTP）
在后端 `.env` 填邮箱服务器信息（用你公司邮箱/腾讯企业邮/阿里邮等）：
```
SMTP_HOST=smtp.exmail.qq.com     # 你的邮件服务器
SMTP_PORT=465                    # 465(SSL) 或 587(STARTTLS)
SMTP_USER=noreply@blitzball.lol  # 邮箱账号
SMTP_PASS=邮箱授权码              # 注意是"授权码"不是登录密码
SMTP_FROM=Blitzball Labs <noreply@blitzball.lol>
```
> 不配 SMTP 也行——序列号照样自动生成，能在后台面板看到，只是不自动发邮件，得你手动复制给买家。

### 5. 重启后端
改完 `.env` 后：`docker compose -f docker-compose.prod.yml up -d`，配置生效。

---

## 配完之后（全自动）

- 用户付款 → 几秒内收到带序列号的邮件。
- 你在面板「序列号」标签能看到每一笔：哪个订单、发了什么码、激活没、绑了哪台机。
- **幂等保护**：Airwallex 偶尔会重发通知，后端认订单号，**同一笔订单只发一个码**，绝不重复。
- 退款了：在面板点「封禁」，那个码立刻失效，用户无法再激活。

---

## 安全说明（已内置）
- Webhook 用 **HMAC-SHA256 验签**，伪造的付款通知会被拒（401）。
- 后端**不接触银行卡号**——卡号只在 Airwallex 收银台，符合合规。
- 只有「付款成功」类事件才发码；「创建/失败」等事件忽略。

---

## 没配支付时也能用（手动发码）
还没接 Airwallex？没关系，现在就能用：
- 面板「序列号」标签 → 选产品 → 点「生成」→ 手动把码发给买家。
- 适合早期少量订单 / 内测 / 朋友购买。

---

## 速查：要填的环境变量
```
# 支付通知
WEBHOOK_SECRET=和Airwallex后台一致的密钥

# 发邮件
SMTP_HOST=  SMTP_PORT=  SMTP_USER=  SMTP_PASS=  SMTP_FROM=
```
都在 `deploy/.env.example` 里有模板。
